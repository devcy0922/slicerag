# 00-L2-common — GoVail 공통 규칙

## 언어와 문서

- 사용자에게 제공하는 설명, 문서, 코드 주석은 한국어로 작성한다.
- 기술 용어, API 필드명, 코드 식별자는 원문을 유지한다.
- README와 아키텍처 문서는 포트폴리오 산출물 수준으로 명확하게 작성한다.

## 보안

- 실제 API Key, DB 패스워드, 내부 토큰, 고객 소스코드는 커밋하지 않는다.
- `.env`는 커밋하지 않고 `.env.example`만 유지한다.
- 테스트 데이터는 `govail-demo-*`, `example.com`, `10.0.0.x` 같은 가상 값을 사용한다.

## 작업 범위

- MVP 범위를 임의로 확장하지 않는다.
- 런타임/멀티에이전트/웹검색은 별도 단계로 분리한다.
- Gateway, Memory, Runtime의 책임 경계를 흐리지 않는다.
